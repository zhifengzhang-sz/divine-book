**作者：** Z. Zhang & Claude Opus 4.6 (Anthropic)

## 工具集

| 工具 | 命令 | 用途 |
|:-----|:--------|:--------|
| `combo-rank.ts` | `bun app/combo-rank.ts --platform X --top N` | 为每个平台（功法）的所有有效组合排名 |
| `combo-cluster.ts` | `bun app/combo-cluster.ts --platform X --top N` | 使用K-means聚类法识别构筑原型 |
| `book-vector.ts` | `bun app/book-vector.ts --platform X --op1 Y --op2 Z` | 生成时间序列因子向量 |
| `book-vector-chart.ts` | `bun app/book-vector-chart.ts --platform X` | 生成HTML图表进行可视化 |

绑定质量（BQ）由 `lib/domain/binding-quality.ts` 内部计算。

### 评分公式

```
原始分 = 技能伤害比率 × 40 + (会心倍率-1)×100×30 + 额外伤害×0.1
    + (同步倍率-1)×100×20 + 治疗削减×5 + 绝对治疗×2 + 绝对减伤×2
最终分 = 原始分 × (0.5 + 绑定质量 × 0.5)
绑定质量 = 利用率 × (0.4 × 平台契合度 + 0.25 × 区域覆盖率 + 0.2 × 区域广度 + 0.15)
```

区域因子定义参见：[domain.category.md](data/domain.category.md)

### 排名注意事项

原始排名会受到"主导锚定词缀"与每个可用伙伴配对的影响而虚高。例如，若【灵犀九重】是最佳锚定词缀，则排名1-5可能全是【灵犀九重】+X的组合——这并非5个独立选择，而是1个锚定选择搭配5个不同的伙伴选项。下方排名在相关处已按锚定词缀进行了分组。

### 模型涵盖范围

| 已涵盖 | 未涵盖 |
|:---------|:-------------|
| 所有链区域中每个神通位的伤害 | 跨神通位的增益/持续伤害传播 |
| 绑定有效性（拒绝无效"幽灵"组合） | 最大生命值百分比伤害（游戏机制） |
| 绑定质量：利用率、平台契合度、区域广度 | 减益层数累积 |
| 时间序列：召唤物、时效性增益、修正器 | 第二轮及后续循环的持续效果 |

## 灵书构造

工具输出的是目标词缀组合。要构造对应的灵书，需要将每个词缀映射到携带它的技能书。

### 灵书结构

每本灵书 = 3本技能书：

| 位置 | 提供 | 决定因素 |
|:---------|:---------|:--------------|
| 主位 | 主词缀（固定） | 始终获得 |
| 辅助位-1 | 从词缀池中随机获得一个副词缀 | 需要"洗练" |
| 辅助位-2 | 从词缀池中随机获得一个副词缀 | 需要"洗练" |

每本辅助技能书的词缀池，按其稀有度排序（出现概率：低 → 高）：

| 类型 | 词缀池 | 稀有度 |
|:-----|:-----|:-------|
| 专属 | 每本技能书唯一对应一个 | 稀有 |
| 修为 | 同一流派内共享（剑/法/魔/体） | 中等 |
| 通用 | 所有技能书共有 | 常见 |

### 词缀 → 技能书映射

| 词缀 | 类型 | 来源技能书 | 流派 |
|:------|:-----|:-----------|:-------|
| 【心逐神随】 | 专属 | `解体化形` | 魔道 |
| 【玄心剑魄】 | 专属 | `春黎剑阵` | 剑修 |
| 【无极剑阵】 | 专属 | `无极御剑诀` | 剑修 |
| 【龙象护身】 | 专属 | `浩然星灵诀` | 法修 |
| 【奇能诡道】 | 专属 | `周天星元` | 法修 |
| 【天倾灵枯】 | 专属 | `甲元仙符` | 法修 |
| 【破釜沉舟】 | 专属 | `十方真魄` | 体修 |
| 【索心真诀】 | 专属 | `惊蛰化龙` | 体修 |
| 【无相魔威】 | 专属 | `无相魔劫咒` | 魔道 |
| 【神威冲云】 | 专属 | `通天剑诀` | 剑修 |
| 【灵犀九重】 | 修为 | 剑修流派技能书 | 剑修 |

剑修流派技能书包括：`春黎剑阵`, `皓月剑诀`, `千锋聚灵剑`, `念剑诀`, `通天剑诀`, `新-青元剑诀`, `无极御剑诀`

技能书的境界（阶数、融合重数、悟境）越高，获得稀有词缀的几率越大。请为每个辅助位使用可获得的最高境界技能书。

### 冲突规则

| 规则 | 触发条件 | 效果 |
|:-----|:---------|:-------|
| 核心冲突 | 两本灵书的主位使用了**同一本**技能书 | 后一个神通位的**整个技能**被禁用 |
| 副词缀冲突 | 两本灵书的辅助位使用了**同一本**技能书 | 后一个神通位的**该副词缀**被禁用 |
| 跨类型重用 | 一本技能书在一个神通位做主位，在另一个神通位做辅助位 | **无冲突**（允许） |

约束：每本技能书在整个6神通位的构筑中，**至多**出现一次作为辅助书。

### 当前模型对此神通位的评估局限

## 神通位 3 —— `甲元仙符`

**最佳组合**： 【龙象护身】+【奇能诡道】 — 评分 420.2，绑定质量 0.43 (587个有效组合)

评分较低是预期内的：此位是增益核心。两个词缀都不直接产生本神通位的伤害。其价值在于跨神通位增益：
- 【龙象护身】将"仙佑"的+70%攻/防/血增益放大至**+280%**，持续12秒（增益覆盖神通位4-6）。
- 【奇能诡道】提供+20%减益层数叠加（为神通位5的真实伤害提供"弹药"）+ 第二轮及之后循环触发的"逆转阴阳"效果。

### 时间序列分析 (`book-vector.ts`)

```
bun app/book-vector.ts --platform 甲元仙符 --op1 龙象护身 --op2 奇能诡道
```

## 已知局限

### 评分方法：标量 vs 向量

当前评分公式将因子向量压缩为一个加权和。这等同对待同区域内加成和跨区域加成，但伤害模型是乘法的：
- 同区域： (1+2) = 3倍 (加法，收益递减)
- 跨区域： (1+1)×(1+1) = 4倍 (乘法，全额收益)

向量化评分目标应将区域因子相乘而非相加——这将自然奖励区域多样性，而无需依赖绑定质量作为代理指标。

### 需要扩展的功能

1.  **神通位5-6分析** —— 解锁后运行排名
2.  **向量化评分** —— 跨神通位的区域因子进行乘法计算
3.  **跨神通位状态模型** —— 追踪在施放窗口后持续存在的增益/持续伤害
4.  **最大生命值%/真实伤害通道** —— 绕过当前词缀伤害链的游戏机制
5.  **第二轮及后续循环建模** —— 跨多个循环周期叠加的持续效果
