# 关键词补充说明

根据《[about.md](../data/raw/about.md)》文档，下面对其中不明确的战斗术语进行系统梳理。通过网络搜索，寻找补充材料和官方解释。遗憾的是，**未能找到由游戏开发商发布的、关于灵书系统伤害计算公式的官方技术文档**。

目前公开的信息主要来自玩家社区和攻略网站，其权威性有限，且大多聚焦于玩法介绍和配置推荐，而非底层公式。不过，在一篇由官方认证账号发布的玩家攻略中，找到了一些关于属性作用的解释，可作为参考。

以下是对四个不明确术语的补充分析与建议：

### 1. “神通加成” (如`千锋聚灵剑`【惊神剑光】)
-   **现状**：文档未明确其属于“攻击力提升”、“伤害加深”还是独立乘区。
-   **分析与推测**：根据其描述“提升神通加成”，且`千锋聚灵剑`本身是多段伤害，此加成很可能是一个**独立于基础攻击力、作用于技能最终伤害倍率的乘数**。在建模时，可将其视为一个独立的“技能伤害倍率提升”乘区。
-   **参考**：玩家攻略([[1]](https://www.87g.com/youxi/181954.html))中提到“功法增伤”等概念，但未直接定义“神通加成”。

### 2. “灵法伤害”的具体对抗属性
-   **现状**：文档未说明是否存在“灵法防御”来专门减免此类伤害。
-   **分析与推测**：在多数手游简化模型中，可能**没有独立的“灵法防御”属性**。“灵法伤害”可能只是一种伤害类型的名称，其减免统一由目标的“伤害减免”、“最终伤害减免”或“守御”属性（如果“守御”是通用防御）来结算。另一种可能是，“守御”属性即通用防御力，直接参与伤害计算中的减法（如 $\text{攻击} - \text{守御}$）。
-   **参考**：未在搜索结果中找到明确说明。

### 3. “守御”属性的具体减伤公式
-   **现状**：完全缺失。
-   **分析与推测**：这是构建精确模型的最大障碍。“守御”可能以两种方式运作：
    1.  **减法防御**：在基础伤害计算中直接扣除，公式可能为 $\text{基础伤害} = \text{攻击力} \times \text{技能倍率} - \text{守御}$（或类似变体）。这是RPG常见设定。
    2.  **百分比减伤**：直接提供一定百分比的“伤害减免”。
    玩家攻略中将其列为重要防御属性，但未给出公式。
-   **行动建议**：**必须通过游戏内实际测试来验证**。例如，记录同一技能对不同“守御”值目标造成的伤害，反推公式。

### 4. 各类“伤害加深”与“伤害减免”的优先级与结算顺序
-   **现状**：文档提到了“神通伤害加深”、“最终伤害加深”、“伤害减免”、“最终伤害减免”等多种乘区，但未说明其叠加规则和结算顺序。
-   **分析与推测**：基于游戏设计常识：
    *   **“最终”类词缀**（如【明王之路】的“最终伤害加深”、【命损】的“最终伤害减免降低”）通常代表**最高优先级的独立乘区**，在计算链的末端生效，与其他增伤/减伤乘算。
    *   **非“最终”类词缀**（如【灵威】的“神通伤害加深”、【金汤】的“伤害减免”）可能**在同一个乘区内加算**，然后再与其他乘区乘算。
