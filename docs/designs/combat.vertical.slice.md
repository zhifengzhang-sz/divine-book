## Vision

### 10x Check
The 10x version is a **live debugging workbench** for combat mechanics. Not just "verify one book matches game data" — a complete parser-to-sim-to-viz traceability chain where you can change a book's effects in YAML, re-run the sim, see the event trace update in real-time with each event annotated by which handler produced it and which raw Chinese text it maps back to. Every verified book becomes a permanent regression test. Coverage tracking tells you which book to verify next.

## Accepted Scope (added to this plan)

- Handler provenance: each emitted event carries `source` field (handler type + effect data)
- Expected vs. actual overlay: input game client numbers, overlay on sim output with red/green diff
- Snapshot test generation: "Lock it in" button generates Bun test file from verified run
- Handler coverage dashboard: which of 91 handlers were exercised, visual progress
- Side-by-side comparison: run two configs, see event traces side by side

## Source Provenance Shape

```typescript
interface EventSource {
  handler: string;    // e.g., "base_attack"
  effectType: string; // e.g., "BaseAttack" (schema type)
  book: string;       // e.g., "斩岳"
}
```

## Verification Criteria

"Verified" means: all damage and healing numbers match game client within 1 unit (integer rounding). Hit count, buff/debuff applications, and state durations must match exactly. For stochastic elements (M_synchro), use seeded RNG for repeatable runs; verify expected values against game client statistical bounds.
